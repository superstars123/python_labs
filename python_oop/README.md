### Теория к сценарию:
**Конструктор `__init__`** — специальный метод, автоматически вызываемый при создании объекта. Он инициализирует атрибуты экземпляра и может проверять корректность входных данных, чтобы объект создавался в корректном состоянии.

![2](/python_oop/images/lab01/2.png) 

### Теория к сценарию:

**Инкапсуляция** — фундаментальный принцип объектно-ориентированного программирования, заключающийся в сокрытии внутренних данных объекта от прямого доступа извне. В Python инкапсуляция реализуется через использование защищенных (с одним подчеркиванием) и приватных (с двумя подчеркиваниями) полей. Это предотвращает некорректное изменение данных и обеспечивает контроль над состоянием объекта.

**Свойства (@property)** — механизм Python, который позволяет управлять доступом к атрибутам объекта. Свойства делятся на три типа:
- **Геттеры** — методы для чтения значений защищенных полей
- **Сеттеры** — методы для установки значений с возможностью валидации
- **Делитеры** — методы для удаления атрибутов (используются реже)

**Валидация данных в сеттерах** — процесс проверки корректности новых значений перед их присвоением. Основные виды проверок:
- Проверка типа данных 
- Проверка диапазона значений 
- Проверка на пустоту 
- Проверка формата 

**Атрибуты класса** — переменные, принадлежащие самому классу, а не его экземплярам. Они являются общими для всех объектов класса и часто используются для хранения констант или общих настроек. Доступ к атрибутам класса возможен как через имя класса, так и через любой экземпляр.

Обработка исключений — перехватывает ошибки и позволяет программе продолжать выполнение без аварийного завершения
![3](/python_oop/images/lab01/3.png)

#### Теория к сценарию:

**Состояние объекта** — совокупность значений всех его атрибутов в конкретный момент времени. Состояние определяет, как объект будет реагировать на вызовы методов. 

**Методы изменения состояния** — специальные методы, предназначенные для перевода объекта из одного состояния в другое. Они обеспечивают контролируемое изменение важных атрибутов, влияющих на поведение объекта.

**Поведение, зависящее от состояния** — ключевая концепция ООП, при которой методы объекта работают по-разному в зависимости от текущего состояния. Это позволяет моделировать реальные объекты, которые меняют свое поведение при изменении условий.
**Обработка исключений (try/except)** — механизм управления ошибками, возникающими во время выполнения программы. Он позволяет:
- Перехватывать исключения определенных типов
- Предоставлять пользователю понятные сообщения об ошибках
- Продолжать выполнение программы после ошибки
- Логировать проблемы для последующего анализа


## Сценарий 4: Сценарий 4: Магические методы и взаимодействие объектов
**Функция в demo.py:** scenario_4_magic_methods_and_comparison()
Демонстрируется работа магических методов и сравнение объектов:

- __repr__ — представление для разработчиков

![4](/python_oop/images/lab01/4.png)

### Теория к сценарию:
Магические методы (dunder methods) — специальные методы Python, определяющие поведение объекта при стандартных операциях
- **`__str__`** — вывод игрока для пользователя
- **`__repr__`** — отладочное представление
- **`__eq__`** — сравнение игроков
- **`__lt__`** — сортировка по уровню и опыту

**Взаимодействие объектов** — объекты одного класса могут сравниваться, сортироваться и использовать общие атрибуты класса
Комплексная валидация гарантирует корректность сравнений и состояний игроков
**Общая теория**
**Класс и объект**
- Класс — шаблон для создания объектов с определёнными атрибутами и методами.
- Объект (экземпляр класса) — конкретная реализация с собственными значениями атрибутов.
- Инкапсуляция - защищает данные объекта от некорректного доступа и изменения, позволяет управлять состоянием через свойства и методы.
- Свойства (@property) позволяют безопасно читать и изменять атрибуты с проверкой корректности.
- Атрибуты класса и экземпляра - класс содержит общие константы (MAX_LEVEL, EXP_TO_LEVEL), экземпляр — индивидуальные значения игрока (_health, _level, _exp).
- Валидация данных - проверяет корректность входных данных при создании и изменении объекта, защищает от ошибок и нарушений бизнес-логики.
