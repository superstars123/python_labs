# Лабораторная работа №1 - Класс и инкапсуляция (Python 3.x)
[![Game Over](https://media1.tenor.com/m/kQb5z-x4qpkAAAAC/game-over-insert-coins.gif)](https://media1.tenor.com/m/kQb5z-x4qpkAAAAC/game-over-insert-coins.gif)
## Тема
**Игровая логика** - класс **Player** (Игрок)
## Цель работы
- Освоить объявление пользовательских классов
- Разобраться с инкапсуляцией (атрибуты экземпляра, закрытые поля)
- Реализовать свойства (@property)
- Переопределить магические методы (__str__, __repr__, __eq__)
- Осознать разницу между атрибутами класса и экземпляра



### Атрибуты (закрытые поля)
| Атрибут | Тип | Описание |
|---------|-----|----------|
| `name` | str | Имя игрока (строка, не может быть пустой)|
| `health` | int | Здоровье игрока (целое число ≥ 0) |
| `experience` | int | Количество опыта (целое число ≥ 0) |
| `level` | int | Уровень (1-10) |
|`active` | bool | логическое состояние (True, если здоровье > 0, иначе False)|


### Атрибуты класса
- `MAX_LEVEL` - макс уровеь 

### Свойства (@property)
| Свойство | Геттер | Сеттер | Описание |
|----------|--------|--------|----------|
| `name` | да | нет | Имя игрока (только чтение) |
| `health` | да | да | Здоровье (нельзя установить < 0)|
| `level` | да | нет | Уровень игрока (только чтение) |
| `experience` | да | нет |Количество опыта (только чтение) |
| `active` | да | нет |True, если игрок жив (health > 0) |

### Магические методы
- `__str__` - красивое описание для пользователя
- `__repr__` - официальное представление для разработчика
- `__eq__` - сравнение объектов

### Бизнес-методы
| Метод | Описание |
|-------|----------|
| `take_damage(damage)` | Наносит урон игроку (здоровье не падает ниже 0) |
| `revive()` | Воскрешает игрока, если он мёртв (восстанавливает здоровье) |
| `gain_experience(amount)` | Добавляет опыт и повышает уровень при достижении 100 EXP |
