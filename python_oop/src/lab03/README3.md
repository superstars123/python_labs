# ЛР-3: Наследование и иерархия классов (Player)

## 1. Цель работы
Освоить механизм наследования классов, научиться строить иерархию объектов, переиспользовать код и переопределять методы. Изучить полиморфизм и создание общих интерфейсов для работы с разными типами игровых персонажей через базовый класс.

---

## 2. Описание реализованной иерархии классов

### Базовый класс Player

Атрибуты:
- _name — имя игрока  
- _health — здоровье  
- _level — уровень  
- _exp — опыт  
- _alive — статус жизни  
- _inventory — инвентарь  

Методы:
- take_damage(dmg) — получение урона  
- gain_experience(value) — получение опыта и повышение уровня  
- revive() — воскрешение игрока  
- calculate_power() — расчёт силы (полиморфный метод)  
- process() — общий интерфейс обработки  
- get_type() — возвращает тип объекта  

---

### Дочерний класс Enemy (Враг)

Атрибуты:
- _damage — урон  
- _rarity — редкость  

Методы:
- attack() — атака врага  

Переопределённые методы:
- calculate_power() — расчёт силы врага  
- process() — вывод боевой мощности  
- __str__() — отображение врага  

---

### Дочерний класс PremiumPlayer (VIP игрок)

Атрибуты:
- _vip_bonus — VIP бонус  

Переопределённые методы:
- calculate_power() — усиленная сила с бонусом  
- process() — VIP обработка  
- __str__() — отображение VIP игрока  

---

### Различия между классами

| Класс | Сила | Особенности |
|------|------|-------------|
| Player | уровень + опыт | базовая прокачка |
| Enemy | урон × уровень | боевой персонаж |
| PremiumPlayer | усиленная + бонус | VIP улучшения |
