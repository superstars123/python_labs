# ЛР-4: Игровая логика (Интерфейсы и абстрактные классы)

## Цель работы
Изучить:
- Абстрактные классы (ABC)
- Интерфейсы как контракты поведения
- Полиморфизм через интерфейсы

---

## Созданные интерфейсы

### 1. Printable
Метод to_string() — возвращает строковое представление объекта (игрока или врага)

### 2. Comparable  
Метод compare_to(other) — сравнивает объекты:
- -1 если текущий слабее
- 0 если равны
- 1 если текущий сильнее

---

## Реализация в классах

| Класс | Printable | Comparable |
|------|----------|------------|
| Player | выводит имя, здоровье, уровень, силу | сравнивает по общей силе |
| PremiumPlayer | добавляет VIP-бонус | сравнивает с учётом бонуса |
| Enemy | выводит тип врага и урон | сравнивает по урону |

## Выводы

1. Интерфейсы задают контракт поведения (что должен уметь объект)
2. Полиморфизм позволяет писать универсальный код
3. Нет необходимости использовать isinstance() в основной логике
4. Разные классы могут по-разному реализовывать один и тот же метод
5. Код становится гибким и расширяемым
